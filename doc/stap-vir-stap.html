Installasie en gebruik van die GEDCOM-na-SAG omskakelaar

1. Op Windows 10

Jy het die Windows Shell for Linux nodig. Besoek hierdie webblad en volg die instruksies. Maak die WSL oop (daar staan bes moontlik iets soos "Bash for Windows"). Jy kry 'n pikswart skerm met iets in die linkerboonste hoek. Hiervandaan werk dit asof jy 'n terminaal oop het op 'n masjien werk waarop Ubuntu geïnstalleer is. Gaan na Stap 2.

2. Op Ubuntu

Jy het die program LifeLines, die verslagprogram sag.ll en jou GEDCOM-lêer nodig. As jy reeds die eerste twee het, spring direk na Stap 5. As jy jou GEDCOM-lêer reeds na 'n LifeLines-databasis omgeskakel het, spring direk na Stap 6.

Maak vir jou 'n omslag waarin jy alles in verband met hierdie program gaan bêre, sê GED2SAG en gaan na hom. Kopieer jou GEDCOM soontoe. Tik ls om te kyk wat het jy nou. Jy moet iets sien soos

…/GED2SAG$ ls
mygedcom.ged
…/GED2SAG$

Telkens hierna gaan ek wys wat op jou skerm staan nadat jy 'n opdrag ingetik het. Alles tot by die dollarteken word deur jou masjien gedoen, ook alles op reëls wat nie sollartekens het nie. Jy tik net telkens in wat hier na die dollarteken staan, gevolg deur Enter (die krom pyl).

3. Installeer LifeLines

Doen die volgende:

…/GED2SAG$ apt-get install lifelines lifelines-doc lifelines-report
…/GED2SAG$ ls /usr/share/lifelines/extract*
/usr/share/lifelines/extract_gedcom.ll  /usr/share/lifelines/extract_set.li
/usr/share/lifelines/extract_html.ll
…/GED2SAG$

Die tweede reëltjie is net om jou gerus te stel dat alles daar is wat daar moet wees.

4. Installeer sag.ll

Doen die volgende:

…/GED2SAG$ git clone http://github.com/dlaurie/ged2sag
Cloning into 'ged2sag'...
remote: Counting objects: 13, done.
remote: Total 13 (delta 0), reused 0 (delta 0), pack-reused 13
Unpacking objects: 100% (13/13), done.
Checking connectivity... done.
…/GED2SAG$ cp ged2sag/scripts/sag.ll .
…/GED2SAG$ ls
ged2sag  mygedcom.ged  sag.ll
…/GED2SAG$

5. Omskep jou GEDCOM in 'n databasis

Doen die volgende:

…/GED2SAG$ llines

Hiervandaan af kom daar 'n aantal menu's op. Party het keuses gemerk met letters by hulle tik jy net die betrokke letter en daar gebeur duidelik iets; ander gee vir jou 'n lyntjie waarop jy 'n aantal karakters gevolg deur Enter moet intik.

Die eerste menu vra: "Enter path". Dit is 'n tegniese manier om te vra wat die naam van jou databasis gaan wees. 'n Goeie keuse is die naam van jou GEDCOM sonder die stertjie.

What directory holds (or will hold) the LifeLines database? (? to list)
enter path:  mygedcom

Aangesien daardie databasis nog nie bestaan nie, kry jy 'n vragie om seker te maak dis wat jy wil hê.

There is no LifeLines database in that directory.
Do you want to create a database there?
enter yes (y) or no (n): n

Nou gaan jy jou GEDCOM inlees. Dit wys etlike opsies, die een wat jy kies is u ("Miscellaneous utilities"). Dit sal 'n submenu laat verskyn.

Please choose an operation: u

Op die volgende menu kies jy R ("Pick a GEDCOM file and read in").

What utility do you want to perform? R

Nou kry jy weer 'n submenu.

Please enter the name of the GEDCOM file. (1/2)
  >1: <Choose outside this list>
   2: mygedcom.ged (06 Feb 2017 - 447Kb) [WikiTree.com]

Tik net 2. daar verskyn 'n skermpie met syfers wat vinnig verander. As dit stop, staan daar:

Jy is nou terug by jou hoofkeuselys. Dit is moontlik om direk hiervandaan die SAG-lêer te maak, maar daar is meer buigsaamheid as jy liewer uitgaan. Druk q totdat jy weer die WSL se swart skerm sien.

6. Uitdruk van jou GEDCOM as SAG

Ons maak eers seker jy het alles.

…/GED2SAG$ ls
ged2sag  mygedcom  mygedcom.ged  sag.ll
…/GED2SAG$

Daar is jou databasis in mygedcom en die uitdrukprogram sag.ll. Ons het nie nou die ander twee nodig nie, maar dis gerusstellend om hulle daar te sien. Ek wil tog benadruk: die GEDCOM self word nie in hierdie stap gebruik nie, as is hy nog daar, net die databasis wat in Stap 5 gemaak is.

Nou kom die uitdruk-opdrag. Ek maak dit doelbewus meer ingewikkeld as nodig; ons kan dit later vereenvoudig.

…/GED2SAG$ llexec -x ./sag.ll -o sag.html -Istart=I1 -Iformat=html -Idepth=100 mygedcom
Program is running...Program was run successfully.
…/GED2SAG$ ls
ged2sag  mygedcom  mygedcom.ged  sag.html  sag.ll
…/GED2SAG$

Daar in sag.html staan jou SAG-geformatteerde uitvoer. Die -I opsies op die uitdruk-reël kan almal weggelaat word, en sal dan die waardes soos hierbo hê. M.a.w. jy kon net ingetik het

…/GED2SAG$ llexec -x ./sag.ll -o sag.html mygedcom

en die resultaat sou dieselfde gewees het. Die -x en -o opsies is egter nodig.
